
OpenAPI Workflow - OpenAPI Spec → Ballerina Connector

USAGE:
  bal tool-id openapi sanitize <spec> <output-dir> [options]
  bal tool-id openapi generate-client <spec> <output-dir> [options]
  bal tool-id openapi fix-code <connector-path> [options]
  bal tool-id openapi generate-tests <connector-path> <spec-path> [options]
  bal tool-id openapi generate-examples <connector-path> [options]
  bal tool-id openapi generate-docs <doc-command> <connector-path> [options]

COMMANDS:
  sanitize          Sanitize and enhance the OpenAPI specification
  generate-client   Generate Ballerina client from sanitized spec
  fix-code          Fix compilation errors in generated client
  generate-tests    Generate mock server + live integration tests
  generate-examples Generate code examples for the connector
  generate-docs     Generate documentation (specify doc-command)

DOC COMMANDS (for generate-docs):
  generate-all
  generate-ballerina
  generate-tests
  generate-examples
  generate-individual-examples
  generate-main

OPTIONS:
  yes      Auto-confirm all prompts
  quiet    Reduce log output

EXAMPLES:
  bal tool-id openapi sanitize /home/user/spec.yaml /home/user/my-connector
  bal tool-id openapi generate-client /home/user/spec.yaml /home/user/my-connector
  bal tool-id openapi fix-code /home/user/my-connector/ballerina
  bal tool-id openapi generate-tests /home/user/my-connector /home/user/spec.yaml
  bal tool-id openapi generate-docs generate-all /home/user/my-connector

---

sanitize

Sanitize and enhance an OpenAPI specification

USAGE:
  bal tool-id openapi sanitize <spec> <output-dir> [options]

ARGUMENTS:
  <spec>        Path to the source OpenAPI specification
  <output-dir>  Directory for sanitized specification files and sanitation records

OPTIONS:
  yes, --yes, -y       Auto-confirm prompts
  quiet, --quiet, -q   Reduce log output

---

generate-client

Generate a Ballerina client project from an OpenAPI specification

USAGE:
  bal tool-id openapi generate-client <spec> <output-dir> [options]

ARGUMENTS:
  <spec>        Path to the OpenAPI specification
  <output-dir>  Directory where the generated Ballerina client is written

OPTIONS:
  yes, --yes, -y       Auto-confirm prompts
  quiet, --quiet, -q   Reduce log output
  remote-methods       Generate client APIs as remote methods
  resource-methods     Generate client APIs as resource methods
  license=<path>       License file path for generated source headers
  tags=<tag1,tag2>     Include only operations from the given OpenAPI tags
  operations=<op1,op2> Include only the given operation IDs
  client-method=<resource|remote>

---

fix-code

Fix Ballerina compilation errors in a generated connector project

USAGE:
  bal tool-id openapi fix-code <connector-path> [options]

ARGUMENTS:
  <connector-path>  Path to the generated Ballerina connector project

OPTIONS:
  yes, --yes, -y       Auto-confirm prompts
  quiet, --quiet, -q   Reduce log output

---

generate-tests

Generate OpenAPI connector tests and mock server module

USAGE:
  bal tool-id openapi generate-tests <connector-path> <spec-path> [options]

ARGUMENTS:
  <connector-path>  Path to the connector workspace
  <spec-path>       Path to the OpenAPI specification

OPTIONS:
  yes, --yes, -y       Auto-confirm prompts
  quiet, --quiet, -q   Reduce log output

---

generate-examples

Generate AI-assisted Ballerina examples for a connector

USAGE:
  bal tool-id openapi generate-examples <connector-path> [options]

ARGUMENTS:
  <connector-path>  Path to the connector workspace

OPTIONS:
  yes, --yes, -y       Auto-confirm prompts
  quiet, --quiet, -q   Reduce log output
  regenerate           Try recovering existing examples before fresh generation

---

generate-docs / generate-all

Generate README documentation for an OpenAPI-generated connector

USAGE:
  bal tool-id openapi generate-docs <doc-command> <connector-path> [options]

DOC COMMANDS:
  generate-all
  generate-ballerina
  generate-tests
  generate-examples
  generate-individual-examples
  generate-main

ARGUMENTS:
  <doc-command>     Documentation mode to run
  <connector-path>  Path to the connector workspace

OPTIONS:
  yes, --yes, -y       Auto-confirm prompts
  quiet, --quiet, -q   Reduce log output
